How can I create Bingo Cards? I want to give it a GN_ID, Number of Songs in the game, and have it generate 200 different bingo cards with the possible winner within a 15 song playlist.
AI
SonicPsyop AI

The approach is a stored procedure that:

  1. Takes @Game_ID, @CardCount (200), and @PlaylistSize (15)
  2. Randomly selects 24 unique songs per card from the pool, laid into the 25 squares (center = FREE)
  3. Guarantees at least one card will have a winning line within the 15-song call sequence
SQL — usp_GenerateBingoCards
CREATE OR ALTER PROCEDURE usp_GenerateBingoCards
	@Game_ID      INT,
	@CardCount    INT  = 200,
	@PlaylistSize INT  = 15
AS
BEGIN
    SET NOCOUNT ON;

    -- Validation
	IF NOT EXISTS (SELECT 1 FROM Game WHERE Game_ID = @Game_ID)
    BEGIN
		RAISERROR('Game_ID %d does not exist.', 16, 1, @Game_ID);
        RETURN;
    END

    -- Build shuffled song pool
    DROP TABLE IF EXISTS #SongPool;
    SELECT
        Song_ID,
        ROW_NUMBER() OVER (ORDER BY NEWID()) AS RandRank
    INTO #SongPool
    FROM Songs
    WHERE Song_IsActive = 1;

	-- Identify "hot" songs called within first @PlaylistSize
    DROP TABLE IF EXISTS #HotSongs;
    SELECT Song_ID
    INTO   #HotSongs
    FROM   #SongPool
	WHERE  RandRank <= @PlaylistSize;

	-- Generate @CardCount cards via WHILE loop
	DECLARE @i INT = 1;
	WHILE @i <= @CardCount
    BEGIN
        -- Reshuffle pool for every card using NEWID()
        DROP TABLE IF EXISTS #Shuffled;
        SELECT Song_ID,
               ROW_NUMBER() OVER (ORDER BY NEWID()) AS Pos
        INTO #Shuffled FROM #SongPool;

        -- Pick 24 songs and map slots to square names
		INSERT INTO #NewCards (CardSeq, Sq_B1, /* ... */ Sq_O5)
		SELECT @i,
            MAX(CASE WHEN Slot = 1  THEN Song_ID END), -- B1
            /* ... slots 2-24 map to B2..O5, skipping N3 ... */
            MAX(CASE WHEN Slot = 24 THEN Song_ID END)  -- O5
        FROM #CardSongs;

		SET @i += 1;
    END

    -- Overwrite Card #1 top row with 5 hot songs (guaranteed winner)
    UPDATE #NewCards
    SET
		Sq_B1 = @Hot1, Sq_I1 = @Hot2, Sq_N1 = @Hot3,
		Sq_G1 = @Hot4, Sq_O1 = @Hot5,
        IsGuaranteed = 1
    WHERE CardSeq = 1;

    -- Bulk insert all cards into Cards table
    INSERT INTO Cards (Game_ID, Card_Date_Create, PlayCount,
                        Card_IsWinner, Card_SeedKey, Sq_B1, /* ... */ Sq_O5)
    SELECT
		@Game_ID, SYSDATETIME(), 0, 0,
        CONVERT(VARCHAR(36), NEWID()),
        Sq_B1, /* ... */ Sq_O5
    FROM #NewCards
    ORDER BY CardSeq;

END;
GO
SQL — usp_PreviewCard (helper)
CREATE OR ALTER PROCEDURE usp_PreviewCard
	@Card_ID INT
AS
BEGIN
    SELECT
        c.Card_ID, c.Game_ID, c.Card_Date_Create,
        sB1.Song_Title + ' – ' + sB1.Song_Artist AS B1,
        sI1.Song_Title + ' – ' + sI1.Song_Artist AS I1,
        'FREE SPACE'                             AS N3,
        -- ... all 25 squares with LEFT JOINs to Songs ...
        sO5.Song_Title + ' – ' + sO5.Song_Artist AS O5
    FROM Cards c
    LEFT JOIN Songs sB1 ON c.Sq_B1 = sB1.Song_ID
    -- ... repeat for all 24 squares ...
	WHERE c.Card_ID = @Card_ID;
END;
GO
SQL — Example Usage
-- Generate 200 cards for Game 1, 15-song playlist:
EXEC usp_GenerateBingoCards
	@Game_ID      = 1,
	@CardCount    = 200,
	@PlaylistSize = 15;

-- Preview the guaranteed winner card:
EXEC usp_PreviewCard @Card_ID = 1;

How the Guaranteed Winner Works

The proc picks 5 "hot" songs from your 15-song playlist and forces them into the top row (B1, I1, N1, G1, O1) of Card #1. Since all 15 songs will be called during the game, that top row is guaranteed to be a complete line before the night is over. Every other card is randomly generated, so real players compete normally.

Key Design Decisions

DecisionWhy
NEWID() re-shuffled per cardTrue randomness — no two cards will be the same
Card_SeedKey stores a GUIDLets you reprint any card identically later
Slot 13 mapped to N4 (not N3)N3 is always NULL (FREE SPACE), so slots skip over it
CREATE OR ALTER PROCEDURESafe to re-run — won't error if proc already exists
Two things to decide before going live:

1. Song pool scoping — Currently pulls all active songs. If you want each GameNight to have its own curated playlist (e.g. 80s Night only plays 80s songs), add a GameNight_Songs junction table and filter there.

2. Duplicate card prevention — With 200 cards and a large song pool, near-duplicates are statistically very unlikely, but if you want to enforce uniqueness you can add a CHECKSUM column on the card layout.
